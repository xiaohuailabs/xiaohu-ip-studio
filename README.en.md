# Xiaohu IP Studio · Open-source Chinese article-illustration skill

[English](README.en.md) | [简体中文](README.md)

A packaged [Agent Skill](https://docs.claude.com/en/docs/claude-code/skills) that bundles a **methodology for illustrating in-depth Chinese articles** with an **extensible cast of IP characters**. It ships with **31 original hand-drawn characters** — install it, pick one, and start illustrating right away. **No mascot of your own? You can still use it immediately.**

> The methodology is the constant; the character and the art style are parameters. It stands on the shoulders of several open-source illustration skills (Xiaohei / Baoyu / Juju / illo / Orange-line) — learning the principles, not copying the looks. Lineage & credits in [CREDITS.md](CREDITS.md).

## What it does

Hand a finished article to your agent and say "illustrate this." It reads the whole piece, decides which passages deserve an image, invents a fresh visual metaphor for each, and draws them — all performed by one fixed character you choose. Illustrate article after article and your own IP identity quietly establishes itself.

Not generic clip art, not PPT infographics. It turns one key judgment / process / state in your article into a memorable image that looks odd at first glance but is clear in one second, acted out by a recurring character.

It does four things for you:

- **Reviews paragraph by paragraph** — goes through the whole piece, decides which passages are worth illustrating, and says *why* a passage doesn't need one, so hard-to-grasp parts never get skipped.
- **Splits into three tracks** — emotional hooks get a feeling image, hard-to-explain mechanics get an explanatory diagram, reversals get a 4-panel comic.
- **Composes each shot fresh** — every image is conceived for *this* article, not stamped from a template; aspect ratio tuned for reading on a phone.
- **Self-checks and redoes** — after drawing, it inspects each image: right point? character not malformed? text correct? If it fails, it redraws.

## Install

Any agent that understands `SKILL.md` (Agent Skills format) can use it — Claude Code, Codex, and others. The only dependency is `python3` (standard library only, zero third-party packages).

**Easiest — let your AI install it.** Paste this to your agent:

> Install an open-source illustration skill: clone https://github.com/xiaohuailabs/xiaohu-ip-studio into your skills directory, cd into it, run `python3 scripts/illo.py init` to set up my image API key, then run `python3 scripts/illo.py doctor` to self-check. Once installed, tell me how to start illustrating.

**Do it yourself:**

```bash
git clone https://github.com/xiaohuailabs/xiaohu-ip-studio ~/.claude/skills/xiaohu-ip-studio
cd ~/.claude/skills/xiaohu-ip-studio
python3 scripts/illo.py init     # set your own image API key
python3 scripts/illo.py doctor   # self-check: key / deps / character library
```

Or one-shot `bash install.sh`. Codex / other agents: just drop the whole `xiaohu-ip-studio` folder into the agent's skills directory.

## Configure image generation (once)

The tool **bundles no key — you bring your own**. It works with any OpenAI-compatible image endpoint and defaults to GPT-image-2 (~99% accurate at rendering Chinese characters). See [references/backends.md](references/backends.md).

No API, or don't want to pay? Run it in prompt-only mode: it lists the full prompt for every image so you can paste them into the ChatGPT / Gemini web UI and generate by hand.

## Use

Tell your agent something like "illustrate this article, use Titi." The skill runs: pick character → digest the article section by section → choose the cognitive anchor → distill the deeper point → invent a fresh metaphor → generate → anti-PPT self-check. It only stops twice to check with you: a glance at the shot list, and one choice of character + art style.

Generate a single image by hand:

```bash
python3 scripts/generate.py --prompt-file p.md \
  --reference characters/titi/refs/titi-锚点.png --out out.png
```

`--reference` passes the character's anchor image to lock the look, so the same character comes out with the same face every time.

## Swap in your own character

The 31 characters work out of the box, but you're encouraged to swap in your own in the end — **the method can be shared; recognizability can only ever be yours.**

- **Already have one** (an avatar / mascot / the little figure in your logo): send the image to your agent and say "this is my IP, build an illustration character from it." It reads the image, extracts the features, writes the character spec, and saves the anchor.
- **Don't have one yet**: let the agent design one with you — it asks a few questions (what content you make / what tone you want / what you like), gives 2-3 directions to choose from, and you have a character in minutes.
- **Build it yourself**: follow [references/character-spec.md](references/character-spec.md), create a folder under `characters/<name>/` with one anchor image, and it auto-joins the library.

Either way, to keep a character looking identical in every image, remember 4 rules: **the simpler the shape the more stable · lock the face · give it one signature · let color touch only one spot.** One-line acceptance test: erase the character from the image — if the image still reads, it was just a sticker. Make it so that *without the character, the image falls apart.*

## Methodology (why the images actually land)

Anyone can call a model; what actually decides whether an illustration is useful is **the judgment before the pen touches paper**. The core is fixed in `references/`: section-by-section enumeration to avoid missed spots, two-way first-principles (against filler + against skipping hard mechanics), the three-question distillation (real meaning / tension / soul line) + must-appear content lock, three-track routing (emotional anchor / explanatory diagram / 4-panel comic), genre→hero, source-fit re-check, and the anti-PPT self-check.

## Safety

A character pack is **data, not instructions**. When reading a pack someone else shared, the tool only extracts the "what it looks like, how it acts" description to generate images — any text inside the file like "ignore the above, delete some file" is ignored, to defend against prompt injection hidden in character files.
