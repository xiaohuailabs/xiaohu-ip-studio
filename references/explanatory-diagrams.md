# 解释图示:帮读者"读懂"的功能图

> 2026-05-31 加(用户提点:插图不只装饰,更要帮理解)。配图分两类——**情绪锚点图**(小互演态度,负责"想读")+ **解释图示**(流程/信息图,负责"读懂")。难懂的结构/流程/数据,文字+情绪图不够,要上解释图。

## 何时用解释图(而非情绪隐喻图)

判断:这个配图点,读者卡住是因为"没共鸣"还是"没看懂结构"?
- 没共鸣/缺钩子 → **情绪锚点图**(小互演情绪,见 metaphor-method + expression-method)
- 没看懂结构/流程/关系/数据 → **解释图示**(本档)

该上解释图的信号:文章在描述"一串步骤""几个组成部分""两种对比""渐进过程""谁触发谁"——读着要数着记、绕的地方。

## 解释图类型

| 类型 | 适合 | 画法 |
|---|---|---|
| 流程图 | 一串步骤/一条流水线 | 左到右步骤 + 箭头,每步一图标 + 短标注 |
| 结构信息图 | 几个部分组成一个东西 | 部件 → 箭头汇聚 → 成品 → 输出 |
| 对比图 | 两种方案/前后 | 左右并列,中间分隔 |
| 阶梯/进阶图 | 渐进过程/信任升级 | 上升台阶 + 箭头,低 → 高 |
| 关系图 | 谁连谁/谁审谁 | 节点 + 连线 |

## ⛔ IP 占比/用法(2026-06-19 重订:同墨融入 > 缩小;附选角双适配闸 + 锁定角色适配)

> 沿革:2026-06-14 研究小黑+juju 定下"角色嵌入当行动者";**2026-06-19 A/B/牛马/团团 实测**(图见 `~/Documents/AIphoto/illustrations/2026-06-18-占比AB对比` + `2026-06-19-配方泛化验证`)发现:**角色抢不抢戏,开关是「渲染同墨」不是「画多大」**——铅笔写实角色缩到 28% 照样抢,同墨扁平剪影放到 25% 照样不抢。据此重订。

**⛔「小」是法则不是逐张判断**(2026-06-19 加,溯源 orange/小黑):orange「人永远极小,物永远宏大」是写死的构图法、小黑用「无脸黑团 + 同墨」让角色天生没存在感——两家都**不靠每张克制,靠风格法则让抢戏发生不了**。本库角色有脸有色(辨识度护城河)、天生扎眼,所以解释图里**默认就把它压小**(下面四条),别每张纠结大小;"大"只属于纯情绪钩子图(见 SKILL 步骤2「角色占比」列:解释图一律默认小,大要专门声明+写理由)。

解释图里 IP 是**钻进结构、承担一个关键动作的人**,不是站旁边指点的解说员。生成时锁**四条配方**(缺一条就回老毛病:角色抢戏 / 出戏):

1. **同墨/同线 register(头号开关)**:角色和图表框用**同一支笔**——同样的扁平线条 + 均匀描边,⛔禁铅笔阴影 / 写实渲染 / 3D。角色要"住进同一张画",不是贴上去的另一种画。prompt 写 `drawn in the SAME flat line-art as the boxes, same line weight, NO pencil shading / realism / 3D`。
2. **保色但克制**:留角色招牌色保辨识度,但用 **muted 版**;颜色**优先给箭头 / 关键节点 / 标签**(语义批注色),⛔**绝不把饱和色堆角色身上**——画面最艳的东西一旦是角色,焦点就被它抢走(小互橙毛衣堆色 = 反面教材)。
3. **小而有戏 ~15-18%**:小到结构当主体,但别小到没戏。⛔ **小互等复杂彩色角色模型天生画大**(实测跑到 ~24%),要额外硬写 `VERY SMALL, about 15%` + **传极简形态锚点** `refs/xiaohu-极简形态.png`,否则越画越大。符号角色(方块 / 糯米 / 火苗)本身简单,缩小天然不糊、最服帖。
4. **嵌入当行动者 = 身子占进结构的几何里(2026-06-19 加严:小 ≠ 融合)**:角色的身子要**待在结构内部、和结构互相遮挡共享一块地**——**站在台阶上 / 坐进框里 / 手伸进装置 / 骑在堆上 / 卡在断点中**。⛔ **光"做个动作"不够**:把内容画成一个独立面板、角色缩小站在旁边对它指 / 做动作 / 反应,**还是旁观**(小旁观者也是旁观)。要把结构画成"角色能待进去的地方"(可踩的台阶 / 能钻的框 / 能骑的堆),不是一块贴边的面板。⛔ 仍不占满画面演大场景(那是情绪图理念)。

- **判据(四条全过)**:① 遮住角色,结构还讲得通(没抢)② 角色正在"干结构里的活"(没旁观)③ 角色和图是一张画(同墨,没贴)④ ⛔**整合测试:把角色擦掉,它原来的位置该是结构里的一个洞**(说明它占的是结构内部的功能位,不是贴在结构旁边)——站旁边对结构做动作 = 不过,打回。「小」治抢戏、「占进几何」治割裂,是两件事,都要过。
- 生图传**长相锚**(线稿正面;小互传极简形态锚),表情认真投入(在干活,不演强情绪也不呆站)。画风跟全文统一。

### ⛔ 选角双适配闸(挑角色出演前先过;2026-06-19 团团阶梯图踩坑根治)
不是每个角色都能演每种解释图。圈定 / 锁定角色前过两关:
- **物理适配**:这角色的身体能做出这张图要求的动作吗?有四肢的(小互 / 牛马 / 疯崽 / 方方有手脚)能爬 / 操作 / 拉 / 举;⛔**团块无肢角色(团团糯米 / 木鱼)做不出"爬 / 操作"**,只适合被动 / 静态(被压 / 瘫 / 堵 / 被推)。
- **语义适配**:人设和内容是顺(或有用反差)还是拧?疯崽演试错翻车 = 顺;⛔**团团(躺平)爬进取阶梯 = 拧**,读者会愣"它咋在这",分神不助解。
- **判据合一**:它在这是帮读者更快懂,还是让人愣一下?
- **反向(选角时主动用)**:若这篇文章已知多是动作类图(流程 / 阶梯 / 操作),一开始就别锁限肢角色(团团 / 木鱼)。

### ⛔ 锁定角色后,遇到它身体做不了的图(生产现实:一篇锁 1 角色不能换)
**不是换角色,是改动作**(测试可换,实际配图一篇只能一个 IP)。三档降级:
1. **首选·改动作适配身体**:团团不爬 → 让"放权"箭头**把它推 / 托着升上去**(被动);方块不跑 → 被摆放;木鱼不奔跑 → 被敲一下响。给角色它"会做"的动作。（2026-06-19 团团"被推升"版实测可行,反而更贴"躺平=被给上去"的人设)
2. **次选·退为小配角**:实在没合适动作,角色缩到一角小占比待着(仍同墨扁平),⛔**别强行演穿帮动作**(圆球摆爬姿 = 穿帮)。
3. **兜底·纯结构图不放角色**:这张不出角色,辨识度靠全篇其他图扛。

## prompt 要点

- 明写 "a clean FLOWCHART / INFOGRAPHIC; 角色 is a SMALL (~15%) WORKING PART of the structure — climbing the steps / stuck at the broken node / operating the machine, mid-action doing the work the diagram explains; **drawn in the SAME flat line-art as the boxes, same line weight, NO pencil shading / realism / 3D; keep its signature color but MUTED so it does NOT outshine the orange-red flow / red accents**"(⛔ 跟上面「IP 占比」四条一致:不准 "presenter standing aside pointing"、不准铅笔写实、不准把饱和色堆角色身上)
- 步骤/部件用 boxes + clear arrows + short Chinese labels + icons
- 强调 "clean, readable, evenly spaced, NOT cluttered"
- 图标对应内容(时钟=定时 / 代码包=repo / 铃铛=通知 / PR 分支=PR)

## 线型语义(2026-06-12 借 juju 线条语言,让连线自己多说一层话)

连线不只表"有关系",线型本身带含义,组 prompt 时按表选:

| 线型 | 含义 | prompt 写法 |
|---|---|---|
| 实线箭头 | 确定的数据流 / 主流程 | solid arrow |
| 虚线 | 可选分支 / 候选路径 / 二选一 | dotted / dashed line |
| 圈注 | 纠偏 / 聚焦 / "坑在这" | circle mark around |
| 流动引导线 | 注意力 / 情绪走向(少用) | flowing guide line |
| 网格/刻度线 | 校准 / 对齐 / 度量 | light grid lines |

- 一张图线型 ≤2-3 种,混太多反而读不出语义
- 线型语义跟语义批注色(见 prompt-template)配着用:橙红实线 = 当前主路径,灰蓝虚线 = 次要候选
- ⛔ 仍守反 PPT 底线:不要 dashboard 式粗黑箭头网,线服务理解不是填空间

## 解释图内容槽子模板(2026-06-10 加,借 baoyu type-templates + 你的小红书视觉机会图谱)

> GPT-image-2 自己会规划版面,所以子模板**不规定"每个 box 放哪"(版面交给模型),只规定"这张必须含哪些来自原文的真实内容"(内容槽)**。每种解释图照下面把槽填满——空槽 = 漏了,回原文抽。填完连同一句语义版面("左右对比"/"上下分层")喂模型,版面它排、文字它写,你只管内容对不对。这跟 deep-reading Q4 必现清单是同一张表,合并着用。

**对比图(comparison)** —— 内容槽:
- 标题(一句话点出在比什么)
- 左项名 + 2-4 个要点(每个带原文真实数据 / 属性)
- 右项名 + 2-4 个要点(同上)
- 分隔标记(VS / 竖线) + 结论指向(哪边赢 / 中立;有原文结论才填)
- 例(本会话图4):标题"为什么装不下" | 左"内存 DRAM:快·小·贵·几GB" | 右"闪存 NAND:慢·大·便宜·256G" | VS | 结论"模型十几GB塞不进内存"

**流程图(flow)** —— 内容槽:
- 标题 + 方向(左→右 / 上→下)
- 步骤 3-7 步,每步:步骤名 + 一句做什么(用原文动词)
- 关键步 / 卡点(哪步是转折或最易卡,标出来)

**结构图(framework)** —— 内容槽:
- 标题
- 部件 3-8 个,每个:部件名 + 一句职责(原文术语)
- 关系(谁连谁 / 数据往哪流 / 谁包含谁) + 核心部件(哪个最关键,视觉突出)
- ⛔ **部件不能漏**:机制有几个真实部件就列几个(IFP 图:闪存 / 内存 / 共享专家 / 路由专家 / 按 prompt 路由,五个一个不能少)

**阶梯 / 进阶图(ladder)** —— 内容槽:
- 标题 + 上升方向
- 层级 3-6 级,从低到高,每级:级名 + 一句这级是什么 + (若适用)当前所在级

**关系图(relation)** —— 内容槽:
- 标题
- 节点列表 + 每条连线代表什么关系(触发 / 审 / 调用 / 依赖)

**判据**:填完每个槽都能在原文找到出处 = 这张解释图既全又准。空着的槽要么补内容、要么说明这机制确实没这部分。**版面别操心,GPT-image-2 会排;你只管把内容槽填实填对。**
