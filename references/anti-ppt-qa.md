# 怎么防止画成 PPT(负向防线 + 自检)

AI 配图默认就会滑向 PPT 和素材库风,必须用密集的"不要什么"把它拽住。本档案是生图后的守门人。

## ⛔ 适用范围(先分清,2026-06-10 加,治"反 PPT 把解释图也反没了")

反 PPT 不是一刀切,分两类图:
- **情绪锚点图 / 隐喻图**:严格反 PPT——别画成冷冰冰图表、别在左上写类型标题、要荒诞要现编隐喻。下面整套"绝对不要"主要管这一类。
- **解释图(流程/信息/对比/架构/时间线)**:目标是**准确 + 直观**,可以有清晰的 boxes / 箭头 / 标注 / 结构——这不算"画成 PPT",这是它的本分。对解释图,**下面这几条不适用**:"不要复杂架构图""不要把每个节点都解释清楚""太 PPT→改成手绘场景"。解释图该把机制的真实部件画全画对(见 deep-reading 必现内容清单 + metaphor 逐一对照),**清楚 > 怪诞**。
- **两类共同守**:反空洞凑图(没料别硬配)、图内不写色值色名/类型标题、中文标注零错字、全篇视觉契约统一。

一句话:反 PPT 反的是"为配而配的空洞图"和"情绪图画僵",**不是反"解释图把事讲清楚"**。

## 绝对不要(注入每条生图提示词的 constraints)

- 不要 PPT 信息图、不要正式流程图、不要课程课件
- 不要商业插画、不要精致扁平插画、不要科技感 UI、不要真实 App 截图
- 不要可爱卡通海报、不要儿童插画、不要表情包
- 不要复杂架构图、不要把每个节点都解释清楚
- 不要复杂背景、渐变、阴影、纸纹、噪点、米色复古纸感
- **不要在左上角写"Workflow / 系统架构图 / 常见坑 / 路线图"等类型标题**
- 不要在图上写出结构类型的名字

## QA 必过项

- **选点复检(source-fit,2026-06-13 借 illo 加,先做这条)**:图画得再好,先回头问一句——**这张图的灵魂话,是不是全文 / 本节真正的修辞任务,还是某个最好画的支撑细节?** 一张完美命中"错锁点"的图,比丑图更坑(读者被带偏还以为懂了)。命中错锁点 → **先回 deep-reading Q0/Q1 重选点,再重生**,别在一个画得很漂亮的错论点上反复磨。判据:对一下 deep-reading 的文体→hero 表,这张图 hero 的东西配不配它的文体。
- **结构完整性(小互别画崩,2026-06-13 借 illo 加)**:小互零件多(眼镜 / 刘海 / 名牌 / 毛衣 / 背带裤 / 狐狸),比简单角色更容易拼错。逐项扫:
  - **穿模**:地平线 / 桌沿 / 货架 / 背景线**不能从小互身体里穿过去**,要在他轮廓处停住(他不透明、挡在前面)
  - **手脚**:数量对、不多长一只手、胳膊不从肚子中间长出来;**一只手最多拿一个道具**,多的放桌上/地上
  - **零件齐且不糊**:红框眼镜在、齐刘海不乱、竖向名牌在、狐狸没糊进背带裤;名牌"小互"二字太小写不清就留**空白黄牌**别画乱码
  - **道具分离**:工具明确握在手里、跟躯干有缝,或放在场景里——不压平贴身上、不从身上长出来
  - 四格 / 多格:**每格单独扫一遍**(缩小重复的小互最容易在这几处漂)
- **IP 占比(小黑式,2026-06-14 研究小黑+juju 根治)**:解释图里角色是**嵌入结构的行动者**(爬台阶/卡断点/操作机器/拉线),不是站旁边的讲解员。⛔ 角色缩角落举牌"指着看" = idle 旁观(打回);⛔ 角色占满画面演场景 = 抢结构(那是情绪图理念,技术文不对)。判据:结构仍是主体(~65-75%)、角色在干结构里的活(~25-35%)。情绪锚点图不受此限(演情绪本就该大)。
- 画幅符合这张按内容定的比例(横向内容才用 16:9,角色/纵向内容偏竖)且 prompt 方向词与之一致、干净白底(若 style-dna 定的是白底)
- 一张图只讲一个核心结构
- 主体不超过画面约 60%,留白充足
- 隐喻成立且是为这篇现编的(没复刻旧案例)
- 画面怪诞、有创意、有意思,不幼稚不卖萌
- 中文标注少、短、能读(最多 5-8 处,每处 2-8 字)
- 颜色克制(配色语义见 style-dna,定调后填)
- (IP 定调后)角色承担核心动作,不只是装饰
- **整组分布达标**(复核分布 gate,完整否决项见 `cognitive-anchors.md`「分布 gate」):情绪锚点图没超过一半;有结构内容的出了解释图;有过程/转折的出了四格漫画——若整组全是"小互+隐喻物件",说明没做图类型分流,打回重分
- **图内文字过关**(借鉴宝玉文字铁律):没有把色值/色名(#hex、"橙红")画成可见标签;关键标注(产品名/数字/步骤名)无错字。错了**改 prompt 减字重生**(新文件名留旧候选),⛔ 不许 PS/SVG/Pillow 涂改;标注必须零错字的解释图可走文字层方案(见 prompt-template)
- **⛔ 代码 / 表格 / 密集专业文字 → 必须示意化,不写满(2026-06-15 g1 错别字根治)**:GPT-image-2 画密集中文 / 代码是**雷区**(实测 g1 翻译官图:销售表→"销售泰"、ORDER→"GRDER"、销售额→"销售飙",一段真 SQL 糊出 5 个错字)。要画代码 / 表格 / 一堆专业标签时:① **代码只画骨架示意**(`SELECT … FROM … WHERE …` 几个词 + `…` 省略,别写完整真代码,宁可"看着像代码"也不追求每个中文字段名都对)② **文件柜 / 表格标签 ≤3 个**,多了精简或干脆留**空白牌** ③ 整张图总文字量压到最低。判据:**图里中文越多越小,错字概率越高——能少写就少写**。
- **示例内容必须 prompt 写死,不许模型即兴造(2026-06-14 疯崽图②对比逻辑散架根治)**:凡图里要出现的**具体示例**(错字 / 数字 / 例句 / 对比项),prompt 必须给出**确定的那串字**,不能只写 "a typo / some numbers" 让模型自由发挥。**左右对比图尤其**:两侧必须**共享同一个写死的锚定对象**(同一句话、藏同一个确定的错字、同一组数),区别只在"谁怎么处理它"——否则模型左右各造一个,对比的锚就散了(图②初版左"材料"右"材科",读者看不出在比什么)。判据:对比图遮住角色,左右两侧的"被对比物"必须是同一个东西。⛔ **这条在 prompt 阶段就守;万一生成后 QA 才发现对比散架,直接重生,绝不"标注小瑕疵"把半成品交付——QA 发现失败信号的唯一动作是重生,不是免责声明 + 甩给用户决定**
- **全篇视觉契约统一**(风格一致复核,2026-06-01):所有图同一背景(白/浅底)、同一光影(柔和平光)、同一景深(清晰无虚化)、同一精致度(盲盒 icon)。任意两张并排看不出风格区别才算过;某张飘成电影感场景 / 景深虚化 → 重生(见 style-dna 视觉契约)
- **同篇张间变化达标**(2026-06-12 借 juju 变化系统,把反复刻做成可勾选):同篇任意相邻两张,至少在 **4 个维度**上不同——①隐喻物件 ②构图(主体位置/视线方向) ③小互表情动作 ④标注位置(见 prompt-template 位置池) ⑤线条/连线语义 ⑥情绪色。**只换标题或只换强调色不算变化**。⚠️ 跟视觉契约不矛盾:契约锁的是渲染层(背景/光影/精致度),变化的是内容层——渲染统一、画面各异才是套图;两张遮掉标题分不出谁是谁 → 重生其一

## 失败信号(命中就重生成或局部编辑)

- 左上角冒出类型标题
- 角色像吉祥物/表情包/可爱卡通
- **情绪图画成"角色站 C 位做夸张表情"(大头贴 / 表情包感)** → 改成"角色陷进体现该情绪的处境 / 物件堆"(演处境 > 演表情,见 metaphor-method;2026-06-15 团团 A/B 实测根治)
- 画面像 PPT、课件、正式流程图
- 元素太多、箭头太多、节点太多
- 文字变成大段解释
- 背景有纸纹/阴影/渐变/米色/噪点
- 中文错字严重或标注读不出
- 太死板,没有荒诞隐喻
- 跟旧案例构图太像
- 同篇两张图只换了标题/换了个色,画面骨架一样(变化维度 <4)
- 颜色只在装饰不在澄清(说不出某个色块的含义 → 去掉它)
- 画外手势/道具纯装饰(没在递交/指向/遮噪/推动 → 删,见 metaphor-method)
- **图很漂亮但画错了重点**:遮住标注问"这图在讲啥",答案不是这节真正的修辞任务 → 选点错了,回 deep-reading 重锁再生,别只调画面(借 illo source-fit)
- **小互画崩**:背景线穿过身体 / 多一只手 / 胳膊根错位 / 道具糊在躯干上 / 名牌乱码 / 狐狸融进衣服 → 重生(这类靠局部编辑修不好,直接重生更干净)

## 迭代修法(对症)

| 症状 | 修法 |
|---|---|
| 太普通 | 让主体成为动作核心,加一个奇怪但成立的隐喻 |
| 太复杂 | 删节点,只留一个动作 + 3-5 个短标注 |
| 表情不对内容 | 按角色包 character.md 映射表换表情,脸和动作要演出这张的情绪(焦虑/得意/心虚/笃定),不能一张呆脸 |
| 太 PPT | 去标题/边框/网格/过多箭头,改成手绘场景 |
| 太像旧案例 | 保留核心意思,换掉主物件和角色动作 |
| 文字错 | 优先局部编辑;错得多就重生成并减少标注数 |

## 交付判据(分两类,2026-06-10 改)

- **情绪 / 隐喻图**:读者先觉得"有点怪",然后 1 秒内看懂——第一眼像教程页 = 不合格。
- **解释图**:读者 1 秒看懂机制,且每个真实部件都对得上原文(回 deep-reading 必现清单逐条核对)。画得清楚利落不丢人,**"怪"不是目标,"准 + 直观"才是**。第一眼看不懂、或机制画错画漏 = 不合格。
