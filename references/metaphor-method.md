# 每张图怎么现编一个隐喻

## 为什么要现编

AI 默认配图会滑向两个深坑:① 像 PPT 信息图 ② 像素材库随便搜的抽象插画。治本办法是**每张图都现编一个生活化的物理隐喻**——把抽象判断变成一个"一看就懂、但有点怪"的具体画面。这是本技能跟普通配图工具的核心差别。

## 八种结构类型(选一种,别混)

| 类型 | 适合 | 画法 |
|---|---|---|
| Workflow 流程 | 输入→处理→输出、AI 工作流 | 左输入、中处理、右输出,橙箭头表主流向 |
| 系统局部 | 数据源/过滤器/渲染器 | 只画 3-5 个核心模块 |
| 前后对比 | 混乱/有序、手动/自动 | 左乱右稳,中间橙箭头,角色更夸张 |
| 角色状态 | 用户痛点、卡住到跑起来 | 2-4 个小状态,每个一句短标注 |
| 概念隐喻 | 内容工厂、脑内黑盒、自动日报 | 一个大怪物件/机器,少量输入一个输出,要有记忆点 |
| 方法分层 | 方法论框架、能力栈 | 一层层盒子(不要正式金字塔),角色在旁搬砖搭建 |
| 地图路线 | 从想法到上线、承接路径 | 一条弯路径,少量节点,角色牵线走路 |
| 小漫画分镜 | 失败到成功、使用前后 | 2-4 格,每格一个动作 |

## 原创隐喻三步法(核心)

每次都从当前文章重新发明,不照搬旧图:

1. **抽象概念 → 一个物理动作**
   卡住、漏掉、变重、分拣、沉淀、发酵、开门、折叠、拆包、回流、称重、接住
2. **系统结构 → 一个低科技物件**
   坏机器、纸箱、抽屉、水管、邮筒、怪表盘、秤、井、梯子、闸门、转盘、黑盒、打孔器、压面机、晾衣绳、怪工位
3. **让小互承担那个动作 + 用表情演出情绪**
   不是站旁边,而是卡在机器里、拉错线、守门、搬运、修补、称重、扶梯子、把东西塞进怪装置。**关键**:小互的表情和动作要贴合这张要讲的情绪(讲焦虑就愁眉叹气、讲得意就昂首、讲心虚就冒汗、讲笃定就挺胸),映射表见 `characters/xiaohu/character.md`。禁止全套一张呆脸

**用时只选 1-2 个物件,别堆满。** 动作要服务核心意思,不要为了怪而怪。

## 一篇一世界(可选,2026-06-12 借 juju)

> 借自 juju-content-illustrations 的"平行世界":视觉契约统一的是**渲染层**(背景/光影/精致度),这条统一的是**叙事空间**——不让一篇 6 张图变成"这张仓库、那张赛道、再一张盖章机"的散装卡片。

**启用条件(两个都满足才用,否则照旧逐张现编)**:
1. 本篇判"配"的图 ≥3 张
2. 多数锚点同属一个机制域(都在讲同一套架构 / 同一条管线 / 同一个产品的不同侧面)

**用法**:先为整篇发明**一个**世界(车间 / 仓库+工作台 / 配电房 / 渡口……从内容来,不从旧图抄),再让每张图 = 这个世界里的**不同认知动作**。小互在同一个空间里干不同的活,读者获得"进入一个现场"的连续感,套图记忆点强一级。

**⛔ 逃生口(准确 > 世界感)**:
- 某个锚点塞进这个世界会比歪(隐喻对照表对不上)→ 那张退出世界、单独现编,宁可散不可歪
- 锚点太异构(一半讲架构一半讲商业)→ 不硬启用,整篇照旧
- 世界本身也走下方「隐喻逐一对照」:世界里每个空间设定要能对应机制的真实部件,纯造景的删

## 画外手势 / 道具(可选构图装置,2026-06-12 借 juju)

画框外伸进来一只手或一件工具——**递交**(把素材递给小互)、**指向**(点出关键位置)、**遮噪**(挡住次要信息)、**推动**(把流程往前推一格)。给画面加"有人在场"的现场感。

- ⛔ **不是每张都放**(juju 自己踩坑后收紧的规则):只在它能干上面四件事之一时出现,纯装饰的"有人伸手"删掉
- 皮肤适配:黑白线稿(B)和混合媒介(G,本来就有"真实物体按住草图"的物理互动)天然合拍;3D 盲盒棚拍契约下慎用,容易破坏干净棚拍感
- 画外元素不抢戏:小互仍是动作主体,画外只是助推

## 隐喻逐一对照(防"比歪",2026-06-10 加)

隐喻最大的坑不是不够怪,是**比歪**——画面生动但跟机制对不上,读者被带偏。现编完隐喻,**逐一对照一遍:隐喻里每个物件 / 动作,精确对应机制的哪个真实部件?写成一张对照表,对不上或对错的就改。**

例(IFP 的仓库/工作台隐喻):

| 隐喻物件 | 对应机制部件 | 对得上吗 |
|---|---|---|
| 又大又慢的仓库 | 闪存 NAND(大、慢) | ✓ |
| 小工作台 | 内存 DRAM(小、快) | ✓ |
| 开工前按整张工单备料 | 按 prompt 路由(不按 token) | ✓ |
| 一直摆台上的常用工具 | 共享专家常驻 | ✓ |
| 临时从仓库搬的工具 | 路由专家按需调入 | ✓ |

- 每个核心物件都能在表里找到精确对应 = 隐喻成立且精准。
- 有物件对不上真实机制(纯为画面好看加的)→ 删或换。
- 有机制关键部件在隐喻里没有对应物 → 补一个,否则图是残的(这正是 deep-reading"必现内容清单"要查的)。
- **终判据**:一个外行看完图 + 标注,能不能把隐喻"翻译"回真实机制?能 → 又准又直观;不能 → 隐喻太绕或对不上,重编。这一条直接决定"图像和内容匹配得准不准"。

## 举例(看怎么从判断到画面)

- "内容要沉淀" → 物理动作:沉淀 → 物件:一口井 → 画面:把内容一桶桶吊进井里,井口冒出来的是金句
- "承接路径" → 不一定画路线,可以画把内容的尾巴接到一个门把手上
- "一鱼多吃" → 不一定画鱼,可以画把一个纸团压成几种形状(压面机)
- "AI 写完拍胸脯说没问题" → 物件:一个盖章机 → 画面:角色给一堆没检查的活疯狂盖"已完成"章,旁边红字"真的吗"

## ⛔ 反复刻规则

`characters/<角色>/refs/` 和 `styles/皮肤样张/` 里的图只用于风格校准(线条密度、留白、配色克制),**不是构图模板**。除非用户明说"照这张改 / 复刻这个",否则:
- 不复用旧构图(传送带断点、漏斗分拣、素材鱼、盖章工具箱、常见坑路径等)
- **同类主题也要换新隐喻**——"承接路径"这次画路线,下次就得换个物件

判据:如果这张图的隐喻在哪篇文章都能用,说明它太通用了,没榨到这篇的独特点,重编。
