# 画风规范:怎么加一个新画风

**画风和角色是两条独立的轴**:同一个角色能换画风,同一个画风能套不同角色。本技能默认锁**手绘线稿**(招牌风)——但你可以加自己的。

## 默认画风:手绘线稿

定义见 `style-dna.md`。一句话:细黑墨线 + 淡彩晕染 + 纯白底 + 手抖随性 + napkin doodle 感。注入 prompt 的就是那段风格块。

## 一个画风 = 一段可注入的 STYLE 提示块

加新画风,本质是写一段"风格描述"插进生图 prompt 的 STYLE 段。照 style-dna 的格式给三样:

- **STYLE 块**:这个画风的核心(笔触 / 材质 / 光影 / 底色)
- **反例**:明确"不要什么"(防模型滑回默认风)
- **配色映射**:角色主色在这个画风下怎么呈现

放进 `styles/<画风名>.md`,在角色 `character.md` 的"默认画风"字段引用它即可。

## ⛔ 一个忠告:别贪多画风

辨识度来自**统一**,不是多样(对标 illo 的 Blot 只锁 riso、小黑只锁黑白线稿)。一套招牌画风让读者一眼认出出品方;每篇换画风 = 稀释辨识度。

要偶尔换画风,优先**临时 prompt 现转**(传锚点图 + "把它改成 X 风"),而不是预生成一堆画风的锚点图囤着——19 角色 × N 画风是组合爆炸,绝大多数用不上,是纯浪费。
